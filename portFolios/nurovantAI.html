Nurovant AI — Web Platform Redesign

Redesign of an AI-assisted learning web platform. The focus was improving clarity of AI output, user control, and predictable navigation so learners can trust what the system generates and act confidently. This is a redesign, not a greenfield build.

Type: Redesign concept Platform: Web Domain: AI + EdTech Focus: Human AI interaction, IA, system feedback

Context

What existed, what was broken, and what the redesign needed to fix.

Problem

The existing experience made learners do too much work to understand what the AI produced, why it produced it, and what to do next. Key issues were unclear hierarchy, weak feedback states, and navigation that did not match learner intent across sessions.

Goal

Redesign the platform so users can generate learning assets with confidence: clear input and output, visible system status, easy correction, and consistent patterns across core learning and AI actions.

My role

What I owned end to end for this redesign.

Ownership

I owned the redesign from audit to flow updates to UI execution: experience mapping, information architecture, AI interaction patterns, component consistency, and high fidelity screens for the core journeys.

Outputs

IA restructure, revised user flows, redesigned key screens, reusable UI patterns for AI states, and interaction notes to guide implementation.

Key areas redesigned

The highest leverage workflows where confusion previously caused drop off.

AI generation workflow

Reworked the generate flow so users always know the current step, what the AI needs, and what they will get. Added clear labels, preview structure, and a consistent action area.

Output review and editing

Designed an output viewer that supports trust and control: structured sections, highlightable content, edit and refine actions, and clear differences between AI output and user edits.

Library and history

Redesigned the library so users can find past sessions and outputs quickly using predictable grouping, filters, and consistent item layouts.

Account and settings

Simplified settings and preferences so users understand model behavior and app defaults. Improved clarity for privacy and usage related options.

Approach

How I made the redesign feel calmer and more reliable.

Audit first

Identified friction points across navigation, AI interaction, and feedback states. Prioritized changes that reduce confusion at decision moments.

Human AI interaction patterns

Designed for control: users can refine, regenerate, and edit without losing context. System messaging explains what is happening and why.

System consistency

Standardized layouts, tables, forms, and action zones so the UI feels predictable across modules. This reduces relearning and supports future scaling.

System states

AI products live or die by feedback, so states were treated as first class UI.

Generation states

  • Generating state that shows progress and sets expectation
  • Success state that confirms what was created and what to do next
  • Failure state that explains what happened and how to recover

Empty and guidance states

  • Empty library states that guide first actions
  • Help text that clarifies input requirements before errors happen
  • Consistent tone for learning context, not generic system copy
